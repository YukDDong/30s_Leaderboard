# Tennis League Board

브라우저에서 바로 실행할 수 있는 정적 테니스 리그 관리 웹서비스입니다.
팀을 직접 등록하고, 단일 풀리그 경기 목록을 생성한 뒤, 팀 간 대진을 교차표로 보면서 원하는 셀을 눌러 모달로 점수를 입력하면 순위표가 자동으로 계산됩니다.
서버, 데이터베이스, 빌드 도구 없이 `HTML + CSS + Vanilla JavaScript + localStorage`만으로 동작하므로 GitHub Pages에 바로 배포할 수 있습니다.

## 1. 프로젝트 소개

이 프로젝트는 테니스 리그 결과를 간단하게 관리하기 위한 정적 대시보드입니다.

핵심 흐름은 아래와 같습니다.

1. 팀 추가
2. 경기 생성
3. 경기 교차표에서 경기 선택
4. 모달에서 점수 입력 또는 수정
5. 자동 순위 계산 확인

점수는 테니스 세트 규칙 해석이 아니라 단순 득점/실점 숫자로 처리합니다.
예를 들어 `6 : 2`를 입력하면 승자는 득점 6, 실점 2로 집계됩니다.

## 2. 주요 기능

- 팀 등록 기능
- 중복 팀명 방지
- 빈 팀명 입력 방지
- 팀 목록 표시
- 경기 생성 전까지만 팀 삭제 허용
- 단일 풀리그 경기 자동 생성
- 교차표 형식 경기 대진표 표시
- 모달 기반 경기 결과 입력 / 수정 / 초기화
- 승 / 무 / 패 자동 판정
- 순위표 자동 계산 및 즉시 갱신
- 총 팀 수 / 총 경기 수 / 입력 완료 경기 수 / 남은 경기 수 표시
- localStorage 저장 및 새로고침 복원
- 전체 데이터 초기화
- GitHub Pages 배포 가능한 정적 구조

## 5. 로컬 실행 방법

빌드 과정이 없습니다.
아래 중 편한 방식으로 실행하면 됩니다.

1. 저장소를 내려받습니다.
2. `index.html`을 브라우저에서 직접 엽니다.
3. 또는 VS Code Live Server 같은 정적 서버로 실행합니다.

정적 파일만 사용하므로 별도 설치가 필요 없습니다.

## 6. 사용 방법

### 팀 추가

1. `팀 등록` 섹션에서 팀명을 입력합니다.
2. `팀 추가` 버튼을 누릅니다.
3. 빈 문자열은 추가되지 않습니다.
4. 이미 등록된 팀명은 중복 추가되지 않습니다.

### 팀 삭제

- `등록된 팀 목록`에서 `팀 삭제` 버튼으로 제거할 수 있습니다.
- 단, 경기 일정이 생성되기 전까지만 삭제가 가능합니다.
- 경기 생성 후에는 데이터 정합성을 위해 팀 삭제가 잠깁니다.
- 팀 구성을 바꾸고 싶다면 `전체 초기화` 후 다시 등록해야 합니다.

### 경기 생성

1. 팀이 2개 이상 등록되어 있어야 합니다.
2. `경기 생성` 버튼을 누르면 단일 풀리그 일정이 생성됩니다.
3. 각 팀은 다른 모든 팀과 1번씩 경기합니다.
4. 예를 들어 9팀이면 총 36경기가 생성됩니다.
5. 이미 생성된 일정이 있으면 중복 생성되지 않습니다.

### 점수 입력

1. `경기 목록 및 점수 입력` 섹션에서 경기 교차표를 확인합니다.
2. 원하는 대진 셀을 클릭합니다.
3. 모달에서 팀 A와 팀 B 점수를 입력합니다.
4. `저장` 버튼을 누르면 결과가 저장되고, 모달이 닫히며 교차표/순위표/진행 현황이 즉시 갱신됩니다.
5. 이미 저장된 경기는 같은 방식으로 다시 열어 수정할 수 있습니다.
6. `결과 초기화` 버튼을 누르면 해당 경기만 미입력 상태로 되돌아갑니다.
7. 모달 바깥 클릭 또는 `ESC` 키로도 닫을 수 있습니다.

### 점수 입력 규칙

- 점수는 `0 이상의 정수`만 허용됩니다.
- 빈 값, 음수, 숫자가 아닌 값은 저장되지 않습니다.
- 잘못된 입력은 모달 내부 인라인 메시지로 안내합니다.

### 순위표 확인

- 경기 결과를 저장하면 순위표가 즉시 다시 계산됩니다.
- 결과를 수정하거나 초기화해도 즉시 반영됩니다.
- 입력 완료된 경기만 집계 대상입니다.

## 7. 순위 계산 기준 설명

집계 항목은 아래와 같습니다.

- 경기수
- 승
- 무
- 패
- 득점
- 실점
- 득실차
- 승점

승점 규칙은 아래와 같습니다.

- 승: 3점
- 무: 1점
- 패: 0점

자동 판정 규칙은 아래와 같습니다.

- `teamA 점수 > teamB 점수` 이면 teamA 승리
- `teamA 점수 < teamB 점수` 이면 teamB 승리
- `teamA 점수 = teamB 점수` 이면 무승부

정렬 기준은 아래 두 가지만 적용합니다.

1. 승점 내림차순
2. 득실차 내림차순

동률이 더 남는 경우 이번 버전에서는 추가 규칙을 적용하지 않습니다.
현재 구현은 팀 등록 순서를 마지막 안정 정렬 기준으로 사용해 같은 조건일 때 화면 순서를 고정합니다.

## 8. localStorage 설명

이 앱은 브라우저의 `localStorage`에 데이터를 저장합니다.
새로고침해도 데이터가 유지됩니다.

사용하는 키는 아래와 같습니다.

- `tennis_league_teams`
- `tennis_league_matches`

저장 범위는 아래와 같습니다.

- 팀 목록
- 경기 목록
- 각 경기의 점수와 입력 완료 여부

기존 localStorage 구조와 키를 그대로 유지하므로, UI가 바뀌어도 기존 저장 데이터는 같은 규칙으로 복원됩니다.
브라우저 데이터를 지우거나 `전체 초기화`를 누르면 저장 내용이 사라집니다.

## 9. GitHub Pages 배포 방법

이 프로젝트는 빌드 없이 바로 GitHub Pages에 올릴 수 있습니다.

1. 이 파일들을 GitHub 저장소 루트에 올립니다.
2. GitHub 저장소의 `Settings`로 이동합니다.
3. `Pages` 메뉴를 엽니다.
4. `Source`를 `Deploy from a branch`로 선택합니다.
5. 배포 브랜치로 `main` 또는 원하는 브랜치를 선택합니다.
6. 폴더는 `/ (root)`를 선택합니다.
7. 저장 후 잠시 기다리면 배포 URL이 생성됩니다.

상대 경로만 사용했기 때문에 추가 설정 없이 `index.html` 기준으로 정상 동작합니다.
정적 파일 구조와 배포 방식은 이번 UI 개편 이후에도 그대로 유지됩니다.

## 초기화 동작 범위

`전체 초기화` 버튼을 누르면 아래 데이터가 모두 삭제됩니다.

- 등록된 팀 목록
- 생성된 전체 경기 일정
- 저장된 모든 경기 결과

초기화 전에 브라우저 확인 창이 한 번 표시됩니다.

## 정책 메모

- 샘플 팀 데이터는 제공하지 않습니다. 처음에는 빈 상태로 시작합니다.
- 경기 생성 후에는 팀 삭제를 막아 데이터 정합성을 유지합니다.
- 서버 없이 로컬 브라우저 저장소만 사용합니다.
- 경기 입력 UX는 교차표 + 모달 방식이지만, 순위 계산 규칙과 저장 구조는 기존과 동일합니다.
